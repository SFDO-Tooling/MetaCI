What's a Merge Freeze?

A merge freeze is a period of time where repositories are restricted to not merging any new code into their main branches. This allows release engineering and the product teams to ensure that no untested changes are inadvertently included in a product version that is released to customers. When a merge freeze is active, all repositories listed below will have pull requests blocked by a failing check in GitHub titled "Merge Freeze". When the merge freeze is lifted, these checks will be reran and will begin passing.
